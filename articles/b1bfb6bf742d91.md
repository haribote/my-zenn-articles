## 設問の定義

始めにクイズの問題を定義します。  
問題と解答の選択肢に加えて入力方式なども決めておきます。  
Reactなどでリストレンダリングするときに `type` プロパティを見て入力項目のテンプレートを出し分けるわけですね。  

```typescript
const questions = [
  {
    name: "Q1",
    legend: "次のうち、世界で最も高い山はどれですか？",
    type: "radio",
    options: ["キリマンジャロ", "マッキンリー", "エベレスト", "富士山"],
  },
  {
    name: "Q2",
    legend: "次のうち、日本の首都はどれですか？",
    type: "select",
    options: ["大阪", "京都", "横浜", "名古屋", "福岡", "東京"],
  },
  {
    name: "Q3",
    legend: "次のうち、海に面している国を2つ選びなさい。",
    type: "checkbox",

  },

```

## 必要なビット数を求める

[基礎編](https://zenn.dev/tor_inc/articles/2d11a5000c1080) で、nビットは $2^n$ 通りの組み合わせを表現できることを解説しました。1ビットで2通りを表せるのですから、 _Yes/No_ で回答するならば入力項目とビットは1対1に対応するわけです。  

では、今回のように3つ以上の選択肢を持つ入力項目はどうすれば良いでしょう？  
まずは設問ごとに組み合わせの数を数えてみましょう。

|設問|入力方式|選択肢の数|組み合わせの数|
|:---:|-----|:---:|-----|
|Q1|ラジオボタン（単一選択）|4|←に「未選択状態」を加えて<br />**5通り**|
|Q2|セレクトメニュー（単一選択）|6|←に「未選択状態」を加えて<br />**7通り**|
|Q3|チェックボックス（複数選択）|4|←の選択肢それぞれに「選択済み/未選択」の<br />2通りを取るので **16($2^4$)通り**|

ご覧の通り、複数選択式のQ3が16通りと最も多くの組み合わせを持っていることが分かります。  
組み合わせの数はすなわち必要なビット数ですから、Q3には **4ビット** 必要ということになります。  
クイズ全体を機械的に扱うには、Q3以外にも同じビット数を割り当ててしまうのがシンプルです。つまり今回の例では、合計で 4ビット x 3問 = 12ビット 必要だということになるのです。

## 新たに必要となるビット演算

まずは [基礎編](https://zenn.dev/tor_inc/articles/2d11a5000c1080) で解説した以下のビット演算について、今回も引き続き必要となりますので一度おさらいしておきましょう。詳しくは基礎編を読んでくださいね。  

- ビットの位置の移動
  - **左シフト**: ビット列を指定された桁の分、左にずらす
- ビット単位の論理演算
  - **ビット論理積（AND）**: 2つのビット列の桁ごとの論理積を求める
  - **ビット論理和（OR）**: 2つのビット列の桁ごとの論理和を求める
  - **ビット排他的論理和（XOR）**: 2つのビット列の桁ごとの排他的論理和を求める

これらに加えて今回新たに必要となるビット演算が **右シフト** です。名前から想像される通り左シフトの反対、すなわち **ビット列を指定された桁の分だけ右にずらす** ものです。  
次節以降ではこれらのビット演算をどのような処理で用いるのかを具体的に説明します！

## 右シフト

ビット列を指定された桁の分、右にずらします。  
ずれた分だけ、下位の桁は失われます。。  

```
  000000110 右シフト 1
= 000000011

  000000110 右シフト 2
= 000000001
```

TS/JSでは `>>` 演算子がその機能を果たします。

```javascript
const x = 6 // 十進数で定義、十進数の 6 は 二進数で 110

console.log(x >> 1) // => 3
// x は内部的に二進数に変換されて 110 => 1桁右シフトする => 結果は二進数で 11 十進数で 3

console.log(x >> 2) // => 1
// x は内部的に二進数に変換されて 110 => 2桁右シフトする => 結果は二進数で 1 十進数でも 1
```
